// what this project does

A Node.js + Express authentication system (Login / Signup) validated with a POM-based Mocha + Chai test suite, integrated into a Jenkins CI/CD pipeline that installs deps, runs unit & integration tests, and publishes HTML reports.

// testing purpose

Automation catches regressions in validation logic and UI behaviour without manual retesting on every commit. POM keeps locators centralized — one change, not twenty.

// jenkinsfile stages
1
checkout
Pull source from Git. Branch: main.
2
install dependencies
Runs npm ci — clean install, reproducible. Faster than npm install in CI.
3
run unit tests
Executes npm run test:unit. Fails stage on any test failure.
4
run integration tests
Executes npm run test:integration. Requires app running or supertest.
5
generate reports
Publishes HTML report via HTML Publisher Plugin. JUnit XML parsed by Jenkins for trend graphs.
// post-build actions
successArchive test reports, mark build green
failurePrint error summary, mark build red
